    • #kata , #dart

    Qu'est-ce que le TDD ? C'est une méthode de développement de logiciel qui pousse l'écriture des tests avant le code du logiciel. L'origine viendrait d'abord du mouvement TFD (Test First Developement), ce qu'apporte le TDD c'est la partie dite de réécriture de code (Refactoring). La technique est assez simple, et se fait en 3 étapes: On écrit un test qui échoue. On écrit le code correspondant pour valider le test.
